# output/parquet/

Purpose:
    Sink location for data written by the Spark Structured Streaming job
    (streaming/spark_stream_consumer.py). This folder is gitignored —
    streaming output is regenerated by re-running the pipeline, not
    committed.

Expected subfolders (created automatically at runtime):
    - rides/         Cleaned, ride-level records (used by Ride Explorer
                      page and as training data for ml/train_model.py)
    - aggregates/     Windowed/aggregated metrics (used by Overview and
                      Streaming Monitor dashboard pages)

Related:
    - CHECKPOINT_PATH in .env points to a separate checkpoint directory
      Spark uses internally for fault tolerance — keep it distinct from
      the data output paths above.
